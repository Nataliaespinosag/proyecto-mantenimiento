El repositorio creado en GitHub contiene un proyecto de desarrollo colaborativo en el cual participan varios integrantes del equipo. Este repositorio fue diseñado con el fin de aplicar conceptos de control de versiones y mantenimiento de software.

Dentro del repositorio se encuentran los siguientes elementos:

Archivos del proyecto funcional trabajado en equipo
Historias de usuario definidas para organizar el desarrollo
Ramas creadas para la implementación de nuevas funcionalidades
Commits realizados por cada integrante del equipo con mensajes descriptivos


El repositorio permite evidenciar el trabajo colaborativo, el control de versiones y la correcta implementación del flujo de trabajo en GitHub. Además, facilita la revisión del progreso del proyecto por parte del docente y garantiza la trazabilidad de cada cambio realizado.
